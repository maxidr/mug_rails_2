Modelo inicial (propuesto)

Paso a paso

Generamos una nueva aplicación

rails new orders

Este comando genera una estructura bajo la carpeta orders

Jquery en vez de prototype

Rails vienen con prototype como framework javascript (en la versión 3.1 utilizará jquery por defecto).

Gracias a un gem vamos a poder cambiar de framework con muy poco esfuerzo, del siguiente modo:

Primero agregamos al Gemfile:

gem 'jquery-rails', '>= 1.0.3'

Luego actualizamos nuestras dependencias ejecutando:

bundle install

Y por último:

rails generate jquery:install --ui

Tenemos listo el entorno de desarrollo. Comenzamos con la aplicación

El producto tendrá, inicialmente, un nombre, una descripción y un precio unitario entonces ejecutamos:

rails g scaffold product name:string description:text price:decimal

Algo similar realizamos para order e item

En este caso, generamos solo el modelo... no queremos un CRUD de este

rails g model item product:references quantity:integer

Ahh de este si queremos todo. Vamos entonces con scaffold para las ordenes

rails g scaffold order description:text

Ehhh quien fue el que paso mal el requerimiento, caso de uso u historia de usuario??? Necesitamos ITEMS en las ORDENES!!!

rails g migration add_items_to_orders

Interesante, la migración nos genera código y podemos utilizarlo como mejor nos guste.

class AddItemsToOrders < ActiveRecord::Migration
 def self.up
    add_column :items, :order_id, :integer
 end

 def self.down
    remove_column :items, :order_id
 end
end

Y completamos la migración (db/migrate/2011....02_add_items_to_orders) con el siguiente comando

rake db:migrate

No tenemos usuarios para esta aplicación. Ayudanos Devise. Agradecer ao José Valim e Carlos Antônio da Silva

gem install devise

Agregamos devise a nuestro Gemfile

rails generate devise:install
rails generate devise user
